> **소요 시간**: 120분 (강의 30분 + 시연 25분 + 과제 45분 + 공유 20분)
> **형태**: Excalidraw 다이어그램 실습
> **산출물**: [노션 데이터베이스](https://www.notion.so/c5672f57243d4b06bb38b99702cec5da)에 Agent 구조 다이어그램 + 발표 가이드
> **참고**: [LangGraph Workflows & Agents](https://docs.langchain.com/oss/python/langgraph/workflows-agents) (코드 없이 구조 시각화 참고)

## 강의 — 30분

### Agent 구조 패턴 3가지

Agent를 설계할 때 구조는 크게 세 가지 패턴으로 나뉜다.
각 패턴은 문제의 복잡도와 판단 빈도에 따라 선택한다.

| 패턴 | 핵심 구조 | 판단 주체 | 적합한 문제 |
|------|----------|----------|------------|
| **자동화형** | 고정된 순차 흐름 | 사전 정의된 규칙 | 반복 업무, 데이터 파이프라인 |
| **분석형** | 조건 분기 트리 | LLM 분석 결과 | 분류, 심각도 판단, 라우팅 |
| **Planner형** | 계획-실행-평가 루프 | LLM 동적 의사결정 | 리서치, 복잡한 다단계 작업 |

### 다이어그램의 구성 요소

다이어그램은 **Node**(처리 단계)와 **Edge**(데이터 흐름) 두 가지로 구성된다.

| 도형 | 의미 | 색상 |
|------|------|------|
| 사각형 | Node — 처리 단계 | 파랑(LLM) / 초록(Tool) |
| 마름모 | 분기 조건 | 노랑 |
| 화살표 (실선) | Edge — 정상 흐름 | 기본 |
| 화살표 (파선) | Edge — 실패/재시도 경로 | 빨강 |
| 둥근 사각형 | 시작점 / 종료점 | 회색 |
| 파선 사각형 | 외부 시스템 (API, DB) | 초록 |

> Excalidraw 사용이 처음이라면 [Excalidraw 사용법 가이드](https://www.notion.so/32b041e36be981d78ca4ee6d0b307800)를 먼저 읽어보세요.

### 색상 규칙

| 색상 | 의미 | 예시 |
|------|------|------|
| 파랑 | LLM 호출 | 요약 생성, 분류, 계획 수립 |
| 초록 | 외부 Tool/API | DB 쿼리, Slack 발송, GitHub API |
| 빨강 | 에러/실패 경로 | 타임아웃, 파싱 실패, 재시도 |
| 노랑 | 분기 조건 | 심각도 판단, 충분성 평가 |
| 회색 | 시작/종료 | 크론 트리거, 사용자 입력, 최종 결과 |

### 실전에서 등장하는 고급 요소

단순한 순차 흐름만으로는 실제 Agent를 표현할 수 없다.
실무 Agent 다이어그램에서 자주 등장하는 요소를 미리 알아둔다.

| 요소 | 설명 | 다이어그램 표현 |
|------|------|---------------|
| **Human-in-the-loop** | 중요 판단 시 사람 승인을 대기 | 특수 Node + 대기 상태 |
| **병렬 처리** | 여러 서브태스크를 동시 실행 | 분기 후 합류 (fork-join) |
| **조건부 재시도/폴백** | 실패 시 대안 경로로 전환 | 빨강 파선 + 대안 Node |
| **상태 체크포인트** | 중간 결과 저장 후 복구 가능 | DB/저장소 연결 표시 |
| **외부 시스템 연동** | API 호출 + 결과 검증 | 파선 사각형 + 검증 Node |

이 요소들을 시연에서 하나씩 다이어그램에 추가하며 확인한다.

## 시연 — 25분

> 강사가 Planner형 Agent를 중심으로, 고급 요소를 하나씩 추가하며 다이어그램을 완성합니다.
> 단순한 루프가 아닌, 실무에서 필요한 구조 요소가 어떻게 다이어그램에 반영되는지 관찰하세요.

### 다이어그램 작성 5단계

모든 다이어그램은 다음 순서로 작성한다:

| 단계 | 작업 | 설명 |
|------|------|------|
| 1 | Input/Output 정의 | 시작점(트리거)과 종료점(최종 산출물)을 먼저 배치 |
| 2 | 핵심 Node 배치 | 주요 처리 단계를 위에서 아래로 나열 |
| 3 | Edge 연결 | Node 간 데이터 흐름을 화살표로 연결 |
| 4 | 분기 조건 추가 | 조건에 따라 경로가 달라지는 지점을 마름모로 표시 |
| 5 | 실패/재시도 경로 추가 | 에러 발생 시 대체 경로를 파선 화살표로 표시 |

### 시연 흐름: Planner형 고급 다이어그램 단계별 구축

**사용 예시**: 기술 조사 보고서 Agent (계획 → 병렬 조사 → 평가 → 보고서)

참고 자료: [Planner형 고급 다이어그램 (노션)](https://www.notion.so/32b041e36be98185baacd64c2fbd8b22)

시연은 다음 순서로 진행한다:

**Step 1 — 기본 뼈대 (5분)**

계획 → 실행 → 평가 → 보고서의 기본 루프를 그린다.
- 시작점: "사용자 입력"
- 종료점: "보고서 전달"
- 핵심 Node 4개: 계획 수립, 정보 수집, 결과 평가, 보고서 작성

**Step 2 — Human-in-the-loop 추가 (5분)**

계획 수립 후 "사람 승인 대기" Node를 삽입한다.
- 왜 필요한가: 비용이 큰 조사를 시작하기 전에 계획을 검증
- 승인/거부에 따른 분기 경로
- 거부 시 계획 재수립으로 돌아가는 루프

**Step 3 — 병렬 처리 추가 (5분)**

정보 수집을 3개 소스로 동시에 분기한다.
- fork: 웹 검색 / GitHub / 논문 검색을 동시 실행
- join: 모든 결과를 수집 후 다음 단계로 합류
- 하나의 소스가 실패해도 나머지는 계속 진행

**Step 4 — 조건부 재시도/폴백 + 체크포인트 추가 (5분)**

평가 실패 시 대안 경로와 중간 저장 구조를 추가한다.
- 정보 부족 시 계획 업데이트 → 재수집 루프
- 최대 반복 초과 시 현재까지 수집한 내용으로 보고서 생성 (폴백)
- 각 반복마다 상태를 체크포인트로 저장 → 장애 복구 가능

**Step 5 — 정리 및 검토 (5분)**

완성된 다이어그램을 전체적으로 검토한다.
- 모든 Node에 색상이 적용되었는가?
- 실패 경로가 빠진 외부 호출은 없는가?
- 무한 루프 방지 조건이 있는가?

### 보조 예시 (참고만)

시연의 메인은 Planner형이지만, 비교를 위해 두 가지 보조 예시도 확인한다:

| 패턴 | 참고 자료 | 핵심 관찰 포인트 |
|------|----------|----------------|
| 자동화형 | [자동화형 다이어그램 (노션)](https://www.notion.so/32b041e36be9810ba375cf01ca50494b) | 모든 Node가 일직선, 분기 없음 |
| 분석형 | [분석형 다이어그램 (노션)](https://www.notion.so/32b041e36be9810691fde9131bc9a849) | 마름모(분기)가 등장, 조건에 따라 경로가 갈라짐 |

### 3가지 패턴 구조 비교

| 구분 | 자동화형 | 분석형 | Planner형 |
|------|---------|--------|-----------|
| 흐름 형태 | 직선 | 분기 트리 | 루프 + 병렬 |
| 분기 조건 | 없거나 1개 | 2개 이상 | 동적 |
| 실패 처리 | 재시도 후 알림 | 심각도별 분류 | 계획 재수립 + 폴백 |
| 고급 요소 | 거의 없음 | 조건 분기 | HITL, 병렬, 체크포인트 |
| 다이어그램 복잡도 | 낮음 | 중간 | 높음 |

## 과제 — 45분

> 개인 Agent 구조 다이어그램을 완성합니다.

### 과제 안내

Session 1에서 도출한 Agent 후보 중 1개를 선택하여 Excalidraw로 구조 다이어그램을 작성한다.
시연에서 본 고급 요소(HITL, 병렬 처리, 재시도/폴백, 체크포인트) 중 자기 Agent에 해당하는 것을 포함한다.

### 제출 방법

1. [노션 데이터베이스](https://www.notion.so/c5672f57243d4b06bb38b99702cec5da)에서 **[템플릿]** 항목을 복제(Duplicate)한다
2. Excalidraw에서 다이어그램을 완성한다
3. 다이어그램을 이미지(PNG)로 내보내어 노션 페이지에 붙여넣는다
4. Agent 개요, 고급 요소 설명, Node 상세, 발표 가이드를 작성한다
5. 작성 완료 후 **상태**를 `제출완료`로 변경한다

### 작성 순서

**1. Input/Output 정의 (5분)**

- 트리거(시작점)를 둥근 사각형으로 배치
- 최종 산출물(종료점)을 둥근 사각형으로 배치
- 두 사각형 사이에 충분한 공간 확보

**2. 핵심 Node 배치 (8분)**

- Session 1에서 분해한 Sub-task를 각각 사각형으로 배치
- LLM 호출 Node는 파랑, 외부 Tool Node는 초록으로 색상 구분
- 위에서 아래로 실행 순서대로 배치

**3. Edge 연결 (5분)**

- Node 간 화살표를 그려 데이터 흐름을 표시
- 화살표 위에 전달되는 데이터를 간단히 기재 (예: `issues: list`)

**4. 분기 조건 추가 (7분)**

- 조건 판단이 필요한 지점에 마름모를 삽입
- 각 분기 경로에 조건 라벨 기재 (예: `심각도 >= 3`, `데이터 유효`)

**5. 고급 요소 추가 (10분)**

아래 중 자기 Agent에 해당하는 것을 최소 1개 이상 포함한다:

| 요소 | 추가 방법 |
|------|----------|
| Human-in-the-loop | 중요 판단 Node 뒤에 "승인 대기" Node 삽입, 승인/거부 분기 |
| 병렬 처리 | 독립적인 서브태스크를 fork-join으로 분기 |
| 조건부 재시도/폴백 | 외부 API Node에 빨강 파선 + 대안 경로 추가 |
| 상태 체크포인트 | 주요 Node 옆에 DB 저장소 파선 사각형 연결 |

**6. 검토 및 보완 (10분)**

- 아래 체크리스트로 자가 검증
- 빠진 Node나 Edge가 없는지 확인

### 필수 요구사항 체크리스트

다이어그램에 다음 요소가 모두 포함되어야 한다:

- [ ] 시작점(트리거)과 종료점이 둥근 사각형으로 표시됨
- [ ] Node가 4개 이상 존재함
- [ ] LLM 호출 Node와 외부 Tool Node가 색상으로 구분됨
- [ ] Node 간 데이터 흐름이 화살표로 연결됨
- [ ] 분기 조건이 1개 이상 마름모로 표시됨
- [ ] 실패/재시도 경로가 1개 이상 빨강 파선으로 표시됨
- [ ] 고급 요소(HITL, 병렬, 재시도/폴백, 체크포인트) 중 1개 이상 포함됨
- [ ] 각 Node에 입력/출력 데이터 타입이 간단히 기재됨

## 공유 — 20분

> 4-5명이 자신의 다이어그램을 발표하고 피드백을 주고받습니다.

### 발표 가이드 (1인 3분)

1. **Agent 소개**: 어떤 문제를 해결하는 Agent인가? (1문장)
2. **패턴 선택**: 자동화형/분석형/Planner형 중 어떤 패턴인가? 왜?
3. **흐름 설명**: 다이어그램을 따라가며 처리 과정 설명
4. **고급 요소**: 어떤 고급 요소를 포함했는가? 왜 그 요소가 필요한가?
5. **실패 처리**: 가장 위험한 실패 지점과 대응 전략

### 피드백 관점

발표를 들으며 다음을 확인하세요:

- 빠진 처리 단계가 있지 않은가?
- 분기 조건이 현실적인가? (너무 단순하거나 너무 복잡하지 않은가?)
- 실패 시 사용자 경험은 어떻게 되는가?
- Human-in-the-loop가 필요한데 빠져 있지 않은가?
- 병렬로 처리할 수 있는 단계가 순차로 되어 있지 않은가?
- 이 다이어그램만 보고 개발을 시작할 수 있는가?
