## 강의 — 25분

> 강사가 MCP와 RAG의 개념, 동작 원리, 적용 기준을 설명합니다.

### 핵심 개념

| 항목 | MCP (Tool 호출) | RAG (문서 검색) |
|------|----------------|----------------|
| 데이터 소스 | API, DB (실시간) | 문서, 매뉴얼 (사전 색인) |
| 동작 원리 | LLM이 Tool 선택 → 실행 → 결과로 답변 | 질문 임베딩 → 유사도 검색 → context로 답변 |
| 강점 | 실시간 데이터, 시스템 조작 | 비정형 지식 검색, 근거 기반 답변 |
| 약점 | 비정형 Q&A에 부적합 | 실시간 데이터 불가 |

### PydanticAI에서의 구현 차이

- **MCP 방식**: `@agent.tool_plain` 데코레이터로 Tool 정의 → `await agent.run()` 호출 시 자동 선택·실행
- **RAG 방식**: 문서 검색 결과를 Agent의 `instructions`에 주입 → Agent가 context 기반 답변 생성

## 시연 -- 25분

> 강사가 MCP 방식과 RAG 방식을 각각 Jupyter Notebook으로 실행하며 동작 원리를 설명합니다.

### 시연 1: MCP(Tool 호출) 방식

`src/demo_mcp.ipynb` 노트북을 열어 셀을 순서대로 실행하세요.

**관찰 포인트:**
- PydanticAI Agent가 질문을 분석하여 어떤 Tool을 호출할지 **자동 결정**한다
- `@agent.tool_plain`으로 정의한 함수의 **docstring이 Tool 설명**이 된다
- 복합 질문 시 **여러 Tool을 연속 호출**하는 것을 확인한다
- Tool 호출 루프를 개발자가 구현할 필요 없이 **PydanticAI가 자동 처리**한다

### 시연 2: RAG(문서 검색) 방식

`src/demo_rag.ipynb` 노트북을 열어 셀을 순서대로 실행하세요.

**관찰 포인트:**
- 문서를 **청크**로 분할하고 **임베딩** 벡터로 변환하는 과정
- 질문과 문서 청크 사이의 **코사인 유사도**로 관련 문서를 검색
- 검색된 문서를 PydanticAI Agent의 **instructions에 주입**하여 답변 생성
- 유사도 점수를 통해 **검색 품질**을 확인할 수 있음

### 두 방식 비교 질문

강사가 두 시연 결과를 비교하며 다음 질문을 던진다:
- "환불 정책" 같은 정적 정보 질문 — 어떤 방식이 더 자연스러운가?
- "내 주문 상태" 같은 실시간 질문 — RAG로 답할 수 있는가?
- 두 방식을 결합하면 어떤 구조가 되는가? (Hybrid 개념 연결)
