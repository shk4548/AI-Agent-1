### 3가지 패턴 비교

| 구분 | 자동화형 | 분석형 | Planner형 |
|------|---------|--------|-----------|
| 핵심 | 반복 실행 | 인사이트 도출 | 자율 계획 |
| 흐름 | 고정 파이프라인 | 입력 -> 분석 -> 구조화 출력 | 계획 -> 실행 루프 |
| LLM 역할 | 요약/변환 보조 | 핵심 분석 엔진 | 계획 + 실행 판단 |
| 예측 가능성 | 높음 | 중간 | 낮음 |

### Pain -> Task -> Skill -> Tool 분석 예시

강사가 "고객 문의 자동 분류 Agent"를 예로 들어 프레임워크를 설명한다.

**1단계: Pain 식별**

```
Pain: CS팀이 하루 200건의 고객 문의를 수동으로 분류하는 데 약 3시간 소요
- 단순 반복 작업이지만 문맥 판단이 필요하여 완전 자동화가 어려웠음
- 분류 오류 시 고객 불만 증가 -> 정확도가 중요
```

**2단계: Task 분해**

```
Task 1: 문의 접수 (이메일, 채팅, 전화 기록 수집)
Task 2: 문의 내용 파악 및 카테고리 분류 (배송/환불/제품문의/기타)
Task 3: 긴급도 판단 (높음/중간/낮음)
Task 4: 담당자 자동 배정
Task 5: 자동 응답 초안 생성 및 발송
```

**3단계: Skill -> Tool 매핑**

| Task | Skill | Tool |
|------|-------|------|
| 문의 접수 | 데이터 수집 | Gmail API, Zendesk API |
| 카테고리 분류 | 텍스트 분류 | GPT-4o-mini (Few-shot) |
| 긴급도 판단 | 감정 분석 | GPT-4o-mini + 규칙 엔진 |
| 담당자 배정 | DB 조회 | 내부 직원 DB API |
| 응답 초안 | 자연어 생성 | GPT-4o (템플릿 기반) |

## 과제 -- 45분

> 개인 과제: 노션에 Agent 후보 2-3개를 Pain -> Task -> Skill -> Tool 형식으로 설계합니다.

### 과제 안내

1. [노션 데이터베이스](https://www.notion.so/190d08cedd074ec1a4f9513f312d2800)에서 **[템플릿]** 항목을 복제(Duplicate)한다
2. 자신의 실제 업무에서 Agent로 자동화할 수 있는 후보 2-3가지를 선택한다
3. 각 후보마다 템플릿 양식에 따라 작성한다 (후보당 1개 항목)
4. 작성 완료 후 **상태**를 `제출완료`로 변경한다

### 작성 형식 (후보당)

```
## Agent 후보: [이름]

### Pain
- 구체적 수치 포함 (소요 시간, 빈도, 비용)
- 현재 어떻게 처리하고 있는지

### Task 분해
| 순서 | Task | 설명 |
|------|------|------|
| 1 | ... | ... |
| 2 | ... | ... |

### Skill -> Tool 매핑
| Task | Skill | Tool |
|------|-------|------|
| ... | ... | ... |

### Agent 패턴
- 자동화형 / 분석형 / Planner형 중 선택
- 선택 이유

### RAG vs Agent
- 외부 시스템 연동이 필요한가?
- 단순 검색인가, 실행(액션)이 핵심인가?
- 결론: RAG / Agent
```

### 진행 순서

1. **후보 도출 (10분)**: 일상 업무에서 반복적이고 시간이 많이 드는 업무 나열 -> 2-3개 선택
2. **상세 설계 (25분)**: 선택한 후보에 대해 위 형식으로 상세 작성
3. **자기 점검 (10분)**: 아래 체크리스트로 완성도 확인

### 체크리스트

- [ ] Pain이 구체적 수치(시간, 빈도, 비용)로 표현되었는가?
- [ ] Task가 5개 이상으로 분해되었는가?
- [ ] 모든 Task에 Skill과 Tool이 매핑되었는가?
- [ ] Agent 패턴(자동화형/분석형/Planner형)이 선택되고 근거가 있는가?
- [ ] RAG vs Agent 판단 근거가 명확한가?
